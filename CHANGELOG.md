# Changelog

## [v0.0.8] 2026-07-04 — 다크 모드 대폭 보정+육아정보 페이지 다크 모드 지원, 글자 크기 5단계, 대시보드 레이아웃 변경

### 1. 다크 모드 잔여 흰색 배경 대폭 보정
- 캡처로 짚어주신 부분 포함, CSS를 전수 조사해서 변수 대신 밝은 회색·파스텔 색을 하드코딩해뒀던 요소들을 찾아 전부 다크 대응:
  - 홈 화면 "오늘의 육아 팁"(광고 자리 대체 콘텐츠, `.ad-fallback`) — 실제 광고 iframe이 아니라 저희가 직접 만든 컴포넌트라 다크 대응이 가능했음 (진짜 Google 광고가 붙으면 그건 저희가 스타일을 못 바꿈)
  - 체크리스트 미완료 항목 행(`.ci`), 정부지원 체크리스트 행(`.gov-cl-item`), 성장 기록 목록 행(`.growth-record-item`)
  - 로그인 화면 탭 배경(`.auth-tabs`)
  - 체크리스트 진행률 바 트랙(`.progress-bar`), 낮은 퍼센트 배지(`.cl-sb-pct-low`), 캘린더 스티커 탭(`.sp-tab`)
- **육아정보 페이지(`guide/`)도 다크 모드 지원 시작** — 지금까지는 앱 본체에서만 되던 걸, 앱 설정 탭에서 고른 값을 육아정보 페이지도 그대로 읽어와 적용하도록 연결함(별도 토글은 없고 앱의 설정을 따라감)
- 관련 코드: `css/main.css`(다크 모드 블록 대거 추가), `guide/guide.css`(다크 모드 블록 신규), `scripts/build-guide.mjs`/`privacy.html`/`terms.html`/`contact.html`(다크 모드 FOUC 방지 스크립트 추가)

### 2. 글자 크기 5단계로 확장, 기본값을 "크게"로
- 기존 3단계(작게/보통/크게 = 15/17/19px)의 "크게"(19px)가 새로운 기본값 "보통"이 되고, 위아래로 2단계씩 넓혀 **매우 작게/작게/보통/크게/매우 크게(15/17/19/21/23px)** 5단계로 재편
- ⚠️ 서비스 초기라 별도 이전 처리는 하지 않음 — 이전에 "작게"를 골라두셨던 분은 이번 개편으로 살짝 커져 보일 수 있음(15px→17px)
- 관련 코드: `js/fontSize.js`, `css/main.css`/`guide/guide.css`(`html[data-fontsize]` 5단계), `css/main.css`(`.fontsize-seg-5` 5버튼 레이아웃)

### 3. 홈 대시보드 카드 순서·레이아웃 변경
- "은유 오늘"과 "체크리스트"가 항상 같은 줄에 나란히 보이도록 카드 순서를 변경(체크리스트를 다음 일정보다 앞으로)
- **데스크톱**: 기존 2열 그리드 그대로라 순서만 바꿔도 자동으로 첫 줄에 나란히 표시됨
- **모바일**: 기존엔 카드가 전부 1열로 쌓였는데, 첫 두 장(은유 오늘·체크리스트)만 2열로 나란히, 나머지(다음 일정·성장 기록·최근 접종·오늘의 육아 팁)는 이전처럼 한 줄씩 차지하도록 변경
- 관련 코드: `js/ui.js`(`renderDashboard` 카드 순서), `css/main.css`(`.dash-grid` 모바일 미디어쿼리)

### 유지된 기능
- 기존 기능·Firebase 스키마 변경 없음 — 전부 스타일/레이아웃/설정값 조정

## [v0.0.7] 2026-07-04 — 설정 탭 너비 조정, 글자 크기 조절 기능, 알림 세부 설정

### 1. 설정 탭 "너비"만 축소 (v0.0.6 오해 정정)
- v0.0.6에서 "설정 탭이 크다"는 피드백을 "설정 탭 안 내용 카드 크기"로 잘못 이해해서 카드를 작게 줄였었는데, 실제로는 **하단 탭 바에서 "설정" 탭 버튼 자체의 너비**가 다른 4개와 똑같이 커서 어색하다는 의미였음
- v0.0.6에서 줄였던 설정 탭 내부 카드·아이콘·글자 크기는 전부 원래 크기로 되돌림
- 대신 `.nav-pills` 그리드를 `repeat(5, 1fr)` → `repeat(4, 1fr) + 0.55fr`로 바꿔서, 홈/캘린더/체크리스트/성장 4개는 기존과 비슷하게 넓게, "설정"만 아이콘 위주로 좁게 표시되도록 함
- 관련 코드: `css/main.css`(`.nav-pills`, `.install-link` 계열 원복)

### 2. 글자 크기 조절 기능 추가
- 설정 탭에 "작게 / 보통 / 크게" 3단 버튼 추가 — 고른 크기가 앱 전체에 즉시 반영되고 `localStorage`에 저장되어 다음에 다시 열어도 유지됨
- **육아정보 페이지(`guide/`)와 정책 페이지(개인정보처리방침/이용약관/문의)에도 동일하게 적용** — 같은 저장값을 읽어서 각 페이지의 `<head>` 인라인 스크립트가 미리 적용해줌(페이지 로드시 번쩍임 없음)
- 앱 본체는 `html { font-size: 17px }`(보통 기준)에 `sm`(15px)/`lg`(19px) 두 단계를 추가하는 방식이라, 대부분의 요소가 자동으로 함께 커지고 작아짐. 캘린더 이벤트 텍스트(`.ev-line`/`.ev-more`)는 이전과 동일하게 이 조절에서 제외됨(px 고정)
- 관련 파일: `js/fontSize.js`(신규), `css/main.css`/`guide/guide.css`(`html[data-fontsize]` 변수), `index.html`/`privacy.html`/`terms.html`/`contact.html`/`scripts/build-guide.mjs`(FOUC 방지 인라인 스크립트)

### 3. 알림 세부 설정 추가
- **카테고리별 on/off**: "오늘 일정 알림"과 "정부지원 마감 알림"을 따로 켜고 끌 수 있음
- **알림 받을 시간대**: 언제든/오전(6~12시)/낮(12~18시)/저녁(18~24시) 중 선택 — 선택한 시간대에 앱을 열었을 때만 알림이 뜸
  - ⚠️ 정해진 시각에 알림을 "보내주는" 기능이 아니라, 앱을 열었을 때 확인하는 기존 방식(포그라운드 체크) 위에서 시간대로 걸러주는 것— 예를 들어 "오전에만"으로 해두고 밤에 앱을 열면 알림이 안 뜨고, 같은 날 나중에 오전 시간대에 다시 열면 정상적으로 뜸
- 관련 코드: `js/notifications.js`(`isCategoryEnabled`, `getTimeWindow`, `isWithinTimeWindow`), `css/main.css`(`.notif-subsettings`, `.notif-check`, `.notif-window-row`)

### 유지된 기능
- 기존 기능·Firebase 스키마 변경 없음 — 전부 localStorage 기반 설정(`momcal_fontsize`, `momcal_notif_cat_today`, `momcal_notif_cat_gov`, `momcal_notif_window`) 추가

- 아주 좁은 화면(380px 이하)에서는 등록 버튼·유저칩과 겹치지 않도록 로고 크기를 살짝만 축소하는 반응형 규칙 추가
- 관련 코드: `css/main.css`(`.logo`, `.brand-mark`), `guide/guide.css`(`.g-logo`, `.brand-mark`), `css/auth.css`(`.auth-logo .brand-mark`)

### 2. "세부 정보" 텍스트를 기존 폰트(Pretendard)로 원복
- 체크리스트 항목을 펼쳤을 때 나오는 상세 설명, 육아정보 페이지의 항목 설명, 개인정보처리방침/이용약관 본문처럼 **정보를 전달하는 긴 글**은 손글씨 폰트(온글잎 박다현체)가 가독성을 해친다는 피드백 — 이 부분들만 Pretendard로 되돌림
- 제목·버튼·로고·짧은 라벨류는 그대로 온글잎 박다현체 유지 (전면 교체가 아니라 "정보 글"만 한정 원복)
- Pretendard를 CDN에서 다시 불러오도록 복원(`@import`)
- 관련 코드: `css/checklist.css`(`.ci-detail`), `guide/guide.css`(`.g-item p`, `.g-doc p/li`), `css/main.css`/`guide/guide.css`(Pretendard `@import` 복원)

### 3. 전체적인 글자 크기 확대 (캘린더 이벤트 텍스트는 제외)
- "전체적으로 글씨가 너무 작다"는 피드백 — 앱 전체(제목·본문·버튼·카드 등)에 쓰이는 `rem` 단위 글자가 다 같이 커지도록 기준 크기(`html { font-size }`)를 16px→17px로 올림

- **캘린더 날짜 숫자(`.day-num`)는 예외 처리하지 않고 다른 곳과 동일하게 커지도록 유지** — "숫자는 같이 키워달라"는 요청 반영
- 관련 코드: `css/main.css`, `guide/guide.css`(`html { font-size: 17px }`), `css/calendar.css`(`.ev-line`, `.ev-more` px 고정)

### 3. 캘린더 날짜 타임존 버그 수정 (중요)
- **버그**: 한국시간 새벽 0시~9시 사이에 캘린더가 하루 전 날짜를 오늘로 표시하는 문제 발견 (예: 7/4 새벽 1시반인데 화면엔 7/3로 표시)
- **원인**: `js/utils.js`의 `today()`가 `new Date().toISOString()`을 썼는데, `toISOString()`은 항상 UTC 기준이라 한국시간(UTC+9)보다 9시간 느림 — 자정부터 오전 9시 사이엔 아직 전날 UTC 시각이라 날짜가 하루 밀려 보였음
- **수정**: `Intl.DateTimeFormat`으로 기기 시간대·설정과 무관하게 항상 한국(Asia/Seoul) 기준 날짜를 계산하도록 변경
- 같은 원인으로 버그가 있던 주간뷰(`renderWeekView`)의 날짜 계산도 함께 수정(로컬 날짜 요소를 직접 조합하는 방식으로 변경) — 월간뷰는 원래부터 다른 방식(`dStr` 직접 조합)이라 문제 없었음
- 체크리스트의 월령 자동 판별 로직(`js/checklist.js`)도 동일 계열의 시간대 오차가 있어 함께 정리(`Date.now()` → `today()` 기반으로 통일)
- 관련 코드: `js/utils.js`(`today`), `js/calendar.js`(`renderWeekView`), `js/checklist.js`

- **해결책**: 형광펜으로 밑줄 치듯, 글자 뒤에 카테고리 색상의 옅은 배경(약 17% 불투명도)만 살짝 깔고, 좌우 여백은 최소(3px)로 줄임 — 박스 느낌은 살리면서도 여백으로 공간을 낭비하지 않음
- 배경은 옅은 톤, 글자는 카테고리의 진한 색 그대로 사용 — 기존에 이미 쓰던 "연한 배경 + 진한 글자" 뱃지 톤(예: 필수 배지 `#FFEBEE` 배경 + `#C62828` 글자)과 같은 규칙이라 브랜드 톤에서 벗어나지 않음
- 배경색 계산은 카테고리 색상 hex 뒤에 알파값을 붙이는 방식(`색상 + '2B'`)이라, 사용자가 색상을 직접 바꿔도(Sprint 21의 `S.evColors`) 옅은 배경이 자동으로 함께 바뀜
- 정부지원 마감임박 항목은 빨간 배경+빨간 글자로 강조(기존과 동일한 로직, 색상만 형광펜 톤으로 적용)
- 관련 코드: `js/calendar.js`(`renderEventLine`, 주간뷰), `css/calendar.css`(`.ev-line`)

### 3. 폰트 원복 (Nunito) — 윤고딕은 적용 불가
- 윤고딕 적용 요청 — 윤고딕은 상업용 유료 폰트라 Google Fonts 같은 무료 CDN으로 가져올 수 없고, 라이선스 구매 후 폰트 파일을 직접 호스팅해야 해서 현재 환경에서는 적용이 불가능함

- 관련 코드: `css/main.css`, `css/auth.css`, `guide/guide.css`, `UI_GUIDELINE.md`

### 4. 서비스 워커 캐시 버전 갱신

## [Sprint 23] 2026-07-03 — 캘린더 필 글자 잘림 추가 수정 (실사용 확인 결과 반영)

### 문제
- Sprint 21에서 필 중복 아이콘(`🟢🟢`)은 없앴지만, 실제 기기로 확인해보니 여전히 "2차 접종 (5종)" → "2차...", "아이돌봄서비스 신청" → "아..." 처럼 한두 글자만 보이고 잘리는 문제가 남아있었음
- **원인**: `.ev-pill`이 `white-space: nowrap` + `text-overflow: ellipsis`로 **한 줄로만** 텍스트를 표시하도록 되어 있었는데, 캘린더 셀 폭이 좁아서(7칸 그리드) 한 줄에 2~3글자밖에 못 들어갔음. 아이콘을 없애 텍스트 자리를 확보했어도, "1줄 말줄임"이라는 표시 방식 자체가 근본 원인이라 여전히 대부분 잘렸음
- 네이티브 캘린더 앱(삼성 캘린더 등)과 비교해보면, 네이티브 앱은 셀이 더 넓고 글자 크기를 작게 써서 한 줄에 들어가는 게 아니라, **여러 줄로 자연스럽게 줄바꿈**해서 표시하고 있었음

### 수정
- `.ev-pill`을 1줄 말줄임 방식에서 **최대 2줄 줄바꿈**(`-webkit-line-clamp: 2`) 방식으로 변경 — "2차 접종 (5종)"이 "2차 접종" / "(5종)" 두 줄로 자연스럽게 보이는 등, 훨씬 많은 글자가 실제로 눈에 보임
- 글자 크기를 살짝 줄이고(`.62rem`→`.58rem`) 필 안쪽 여백도 줄여서(`3px 6px`→`2px 5px`) 좁은 셀 안에서 텍스트가 들어갈 공간을 최대한 확보
- 2줄 텍스트가 들어갈 수 있도록 캘린더 셀 높이를 늘림 (`min-height: 80px→96px`, 모바일 `52px→76px`) — 이전엔 모바일에서 셀이 오히려 더 작아서(52px) 문제가 더 심했음
- 주간뷰도 같은 `.ev-pill` 클래스를 공유하므로 자동으로 함께 개선됨 (별도 코드 수정 불필요)
- 그래도 다 못 담는 아주 긴 제목(예: 정부지원 제도명)은 2줄 이후 말줄임 처리되지만, 날짜를 탭하면 아래 날짜 패널에서 전체 텍스트를 항상 확인할 수 있음 (기존과 동일)
- 관련 코드: `css/calendar.css`(`.ev-pill`, `.cal-cell`) — JS 변경 없음, 순수 스타일 수정

### 유지된 기능

- **수정**: 필수 항목이 100% 완료되기 전에는 선택 항목이 점수에 전혀 반영되지 않도록 변경. 필수 100% 달성 후에만 선택 완료율(0~100%)이 그대로 보너스로 더해지며, 최대 200%까지 오르도록 상한을 150%→200%로 변경

  - 육아: "현재 XX개월" (3개월 미만은 주 단위)

  - 🏅 **Perfect** (100%): 필수 항목 모두 완료 — 금색 진행바
  - 👑 **Master** (120%): 필수 완료 + 선택 40% 이상 — 보라 진행바
  - 🌈 **Legend** (150%): 필수·선택 모두 완료 — 레인보우 애니메이션 진행바
- **육아 카테고리 라벨 개선** — 월령 범위 명시 (예: "2개월" → "2~3개월")

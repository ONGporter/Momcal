### 1. 성장그래프 — 오늘 날짜가 축의 70% 지점에 오도록 조정
- 기존엔 "마지막 기록"이 축 경계 근처에 딱 맞게만 계산되어, 최근 한동안 기록을 안 남기면 그래프가 옛날에 머물러 보였음
- 오늘 기준 우리 아이의 생후 일수가 축의 약 70% 지점에 오도록 축 범위를 계산 (예: 오늘 140일 → 0~200일 축)
- 여전히 100일 단위로 반올림해서 보기 좋게 확장됨 (100→200→300…)
- 관련 코드: `js/growthChart.js` (`computeAxisMax`에 오늘 날짜 기준 계산 추가)

### 2. 캘린더 ↔ 체크리스트 완료 상태 양방향 연동
- 체크리스트의 "DTaP 1차", "영유아 건강검진 1차" 같은 항목과 캘린더의 같은 이름 일정은 지금까지 완전히 별개로 관리되어, 같은 걸 두 번 체크해야 했음
- 이제 한쪽에서 체크하면 다른 쪽도 자동으로 완료 처리됨 (양방향)
- 이름이 명확히 대응되는 예방접종·건강검진 항목 약 28개를 매핑 (`data/checklist-links.js`) — 발달 체크 같은 캘린더에 대응 일정이 없는 항목은 연동 대상 아님
- ⚠️ 연동 기능 도입 **이전**에 한쪽만 체크해뒀던 기존 데이터는 자동으로 맞춰지지 않음 (새로 체크/해제하는 시점부터 연동)
- 신규 파일: `data/checklist-links.js`, `js/checklistCalendarLink.js`
- 변경 파일: `js/checklist.js`(`tgCk`에 연동 훅), `js/calendar.js`(`saveEventMod`에 연동 훅, 순환 import 방지를 위해 window 경유 호출)

### 4. PWA 홈 화면 추가

### 6. 예방접종 묶음 필의 꾹 눌러 이동 버그 수정

- 관련 코드: `js/calendar.js`(`moveEvent`가 단일 인덱스·배열 모두 처리하도록 변경, 그룹 필에 드래그/터치 속성 복원)

## [Sprint 7] 2026-07-02 — 체크리스트 진행률 버그 수정

사용자가 실제 사용 중 발견한 5가지 버그를 수정했습니다.

### 버그 1 — 카테고리 클릭 시 하이라이트 색이 안 바뀜
- **원인**: 사이드바 항목 클릭 시 `renderClMain()`만 호출되어 사이드바 자체는 다시 그려지지 않았음 → 체크리스트 항목을 체크해야(그때는 `renderClSidebar()`가 호출됨) 비로소 색이 바뀌는 것처럼 보였음
- **수정**: 카테고리 클릭 시 `renderClSidebar()`를 호출하도록 변경 (`js/checklist.js`)

### 버그 2 — 진행률 0~49%도 항상 녹색으로 표시
- **수정**: `basePct < 50`일 때 회색 스타일(`cl-sb-low` / `cl-status-low`)을 적용하도록 CSS 추가 (`css/checklist.css`). 50% 이상은 기존 녹색 유지, 100% 이상은 기존 금색/보라/레인보우 티어 색상 유지

### 버그 3·4 — 필수·선택 구분 오류 + 상한 200%로 변경
- **원인**: 기존 `calcScore` 공식이 필수 완료 여부와 무관하게 선택 항목 점수를 항상 더했음 (`optDone/optTotal*50`을 무조건 가산) → 예를 들어 2~3개월 카테고리(필수 6·선택 2)에서 필수 4개 + 선택 2개만 체크해도 구버전 점수가 117%로 계산되어 "필수 100%" 배지가 잘못 표시됨
- **수정**: 필수 항목이 100% 완료되기 전에는 선택 항목이 점수에 전혀 반영되지 않도록 변경. 필수 100% 달성 후에만 선택 완료율(0~100%)이 그대로 보너스로 더해지며, 최대 200%까지 오르도록 상한을 150%→200%로 변경
  - 예) 필수 4·선택 2 기준: 필수 3+선택 1 = 75% (선택 무시) / 필수 4+선택 0 = 100% / 필수 4+선택 1 = 150% / 필수 4+선택 2 = 200%
  - 티어 판정도 점수 임계값이 아닌 실제 완료 개수(`reqDone/reqTotal/optDone/optTotal`) 기준으로 변경해 계산 오차 여지를 없앰
  - 체크/해제는 항상 현재 `checks` 상태 전체를 다시 계산하는 방식이라, 체크한 순서나 이전 이력과 무관하게 항상 지금 체크된 항목 기준으로 정확한 값이 나옴 (토글 온/오프 반복 테스트로 검증)

### 버그 5 — Perfect/Master/Legend 배지가 2번 깜빡임
- **원인**: `tgCk()`에서 `renderClMain()`을 먼저 호출한 뒤, 그 안에서 호출하는 `renderClSidebar()`가 내부적으로 다시 `renderClMain()`을 호출해 같은 틱에 메인 영역이 2번 그려짐 → 배지의 페이드인 애니메이션이 2번 재생됨
- **수정**: `tgCk()`에서 `renderClSidebar()`만 호출하도록 정리 (사이드바 %와 메인 화면이 한 번에 갱신됨)

- **홈 대시보드 6종 카드** — 홈 화면 상단(가족 프로필 아래)에 "오늘의 대시보드" 섹션 신설
  - 👶 **오늘 며칠째 / 임신 주차** — 출생일 기준 D+일수·개월 수, 또는 임신 주차·출산까지 남은 주수
  - 📅 **다음 일정** — 자동/커스텀 일정을 통틀어 가장 가까운 미완료 일정과 D-day, 탭하면 캘린더로 이동
  - 📋 **오늘 체크리스트** — 현재 주차·월령에 해당하는 체크리스트 카테고리의 완료 개수, 탭하면 체크리스트로 이동

  - ⭐ **오늘의 육아 팁** — 날짜 기준으로 매일 자동 순환되는 육아 팁 (전체 폭 카드)

  - Firestore(`users/{uid}.growthRecords`)에 저장 — 추후 성장그래프 탭에서 동일 데이터 재사용 예정
- **오늘의 육아 팁 데이터** — `data/tips.js`, 30개 팁을 연중 일수 기준으로 결정적 순환 (Firebase 미사용, 정적 콘텐츠)

## [Sprint 3] 2026-07-01 — 체크리스트 개편

### 추가 기능

- **임신 주차별 세분화** — 3분기 → 9개 주차 범위 카테고리로 재구성
  - 4~7주 / 8~11주 / 12~15주 / 16~19주 / 20~23주 / 24~27주 / 28~31주 / 32~35주 / 36~40주
  - 각 주차마다 의학적으로 적절한 필수·선택 항목 배치
- **월령 자동 선택** — 아이 등록 정보 기반 현재 주차·월령에 맞는 카테고리로 자동 이동
- **컨텍스트 배너** — 체크리스트 상단에 현재 주차·분기 또는 월령 표시
  - 임신: "현재 XX주차 · N분기 · 출산까지 약 YY주"
  - 육아: "현재 XX개월" (3개월 미만은 주 단위)
- **진행률 티어 시스템** — 필수 100% 달성 후 선택 항목으로 초과 달성
  - 점수 = 필수 0~100% + 선택 0~50% = 최대 150%
  - 🏅 **Perfect** (100%): 필수 항목 모두 완료 — 금색 진행바
  - 👑 **Master** (120%): 필수 완료 + 선택 40% 이상 — 보라 진행바
  - 🌈 **Legend** (150%): 필수·선택 모두 완료 — 레인보우 애니메이션 진행바
- **육아 카테고리 라벨 개선** — 월령 범위 명시 (예: "2개월" → "2~3개월")

### 변경된 파일

- `data/checklist-data.js` — 임신 카테고리 재구성 (preg_1tri/2tri/3tri → preg_w04~w36)
- `js/checklist.js` — autoSelectCat, switchClTab, calcScore, getTier, 배지 렌더링 추가
- `css/checklist.css` — 티어별 진행바·배지 스타일, 컨텍스트 배너 스타일 추가
- `index.html` — `#clContextBanner` div 추가

### 데이터 구조 변경

- 임신 체크리스트 카테고리 키 변경: `preg_1tri/2tri/3tri` → `preg_w04/w08/.../w36`
  - ⚠️ 기존 임신 체크 데이터(preg 탭)는 Firestore에 유지되나 새 카테고리에 미반영
  - 육아/이유식 카테고리 키(`m0`, `m2`, `f6` 등) 및 아이템 ID 유지 → 기존 체크 데이터 정상 동작

### 🚨 가장 먼저 확인해주세요 — 가족 그룹 공유, Firestore 보안 규칙 추가 필요
v0.0.12에서 "가족 그룹으로 공유(베타)" 기능을 추가했는데, **Firebase 콘솔에서 보안 규칙을 직접 추가해주셔야 실제로 동작합니다.** 콘솔 → Firestore Database → 규칙에서 기존 `users/{uid}` 규칙은 그대로 두고 아래 블록만 추가해주세요:
```
match /families/{familyId} {
  allow read, write: if request.auth != null;
}
```
이걸 안 하면 "가족 그룹 만들기"를 눌러도 저장이 실패하거나, 초대 코드로 참여가 안 될 수 있어요. 자세한 설계는 `PROJECT_SPEC.md`의 "가족 그룹 공유" 섹션 참고.

### 미완성 — 다음에 이어서 해야 할 것
1. **가족 그룹 공유는 베타 단계** — 실제 두 계정으로 테스트는 아직 안 해봄(보안 규칙 추가 후 옹짐꾼님이 직접 테스트 필요). 문제 생기면 캡처/에러 메시지로 알려주세요
2. **한국 공휴일 데이터는 2025~2028년까지만 등록됨** (`data/kr-holidays.js`) — 2029년 이후는 매년 말 다음 해 공휴일이 확정되면 추가해야 함. 2028년은 설날·추석 등 음력 날짜는 맞지만 대체공휴일 일부는 아직 확정 고시 전이라 보수적으로 비워둔 상태
3. **FCM 진짜 푸시 알림** — 아직 "앱을 열었을 때 확인하는" 방식(`js/notifications.js`)뿐, 앱을 꺼도 오는 진짜 푸시는 아님. 완성하려면 옹짐꾼님이 Firebase 콘솔에서 직접 해야 하는 일(Cloud Messaging 사용 설정+VAPID 키 발급, Blaze 요금제 전환)이 있음

### 알아두면 좋은 것
- 정식 도메인은 `momcal.app` — `momcal.vercel.app`도 301 리다이렉트로 계속 접속은 가능(단, 서비스워커 이슈로 예전 방문자는 갱신 안 될 수 있음, 감수하기로 결정됨)
- 폰트 2종 체제: 제목/본문/버튼/로고는 "온글잎 박다현체", "정보 글"은 "오뮤 다예쁨체" — 둘 다 `font-weight: normal` 고정
- 하단 탭: 홈/캘린더/체크리스트/성장/설정 5개, "설정" 탭만 폭이 좁게 디자인됨
- 체크리스트 실시간 동기화(onSnapshot) 관련 코드를 만질 땐 `js/state.js`의 `subscribeToUserData` 주석 참고 — `hasPendingWrites`로 "내 저장 에코"와 "실제 원격 변경"을 구분해야 화면이 불필요하게 중복 렌더링되지 않음
- 캘린더 주간 뷰는 `S.calWeekRef`(YYYY-MM-DD) 자체 기준일을 갖고, `calMove()`가 뷰에 따라 분기(월간=월 단위, 주간=7일 단위)해서 이동시킴 (`js/calendar.js`)
- 캘린더 셀에서 이유식 스티커(`FOOD_STICKER_SET`)는 다른 스티커의 "+N" 카운트와 완전히 분리되어 날짜 숫자 옆에 표시됨
- 캘린더 토·일·공휴일 날짜는 `--holiday-red` 변수로 붉은색 표시(`data/kr-holidays.js`로 공휴일 판별) — 공휴일 이름 자체는 화면에 안 띄움(TODO 아래 "고민 중" 항목 참고)
- **가족 그룹 공유(v0.0.12)**: `users/{uid}.familyId`가 있으면 `families/{familyId}` 문서를 데이터 소스로 씀. 자세한 구조는 `PROJECT_SPEC.md` "가족 그룹 공유" 섹션 참고. 가족 그룹 생성/참여/탈퇴는 전부 `location.reload()`로 마무리함(가족 전환 시 캐시된 파생 상태가 꼬이는 걸 막기 위한 의도적 선택)

### 고민 중 — 아직 결정 안 됨
- [ ] **공휴일 이름을 날짜 옆에 표시할지** — 이유식 스티커랑 자리가 겹칠 수 있어서 옹짐꾼님이 고민 중. 지금은 공휴일 날짜 숫자만 붉은색으로 표시하고 이름은 표시 안 함
- [ ] **스플래시 화면 추가** (v0.0.14에서 문의) — 두 가지 방식이 있음:
  1. PWA 표준 스플래시(OS가 manifest.json 기반으로 자동 생성) — 이미 `manifest.json`의 `name`/`background_color`/`theme_color`/아이콘이 갖춰져 있어서 안드로이드는 추가 작업 없이 어느 정도 됨. iOS는 기기별 `apple-touch-startup-image` 이미지를 여러 장 등록해야 완벽해짐(번거로움)
  2. **앱 자체 스플래시**(추천) — `index.html`이 로드되자마자(Firebase/JS 로딩 전에) 로고+로딩 애니메이션이 담긴 전체화면 div를 먼저 보여주고, 앱 초기화(로그인 확인, 데이터 로드)가 끝나면 부드럽게 사라지게 하는 방식. OS나 브라우저에 따라 들쭉날쭉한 PWA 표준 스플래시보다 훨씬 일관되게 보이고, 지금 있는 다크모드 FOUC 방지 스크립트와 비슷한 패턴이라 구현 난이도도 낮음
  - 진행하게 되면 로고 파일(정사각형, 투명배경 PNG 권장)이 필요함

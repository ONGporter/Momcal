### 구글 로그인 `redirect_uri_mismatch` 에러 — 원인 파악 중
momcal.app에서 구글 로그인 시도 시 400 에러(`redirect_uri_mismatch`) 발생. `js/firebase.js`의 `authDomain`은 그대로 `momcal-fd12b.firebaseapp.com`이라, 실제 구글에 전달되는 redirect_uri는 항상 `https://momcal-fd12b.firebaseapp.com/__/auth/handler`임 (momcal.app 여부와 무관). 즉 momcal.app 자체가 원인이 아니라, GCP에서 momcal.app용 리디렉션 URI를 추가하던 중 원래 있던 `https://momcal-fd12b.firebaseapp.com/__/auth/handler` 항목이 실수로 지워졌을 가능성이 높음. 확인할 것:
- [ ] GCP 콘솔 → Google Auth Platform → Clients → Firebase 자동 생성 Web client → **승인된 리디렉션 URI**에 `https://momcal-fd12b.firebaseapp.com/__/auth/handler`가 남아있는지 확인, 없으면 재추가
- [ ] Firebase 콘솔 → Authentication → Sign-in method → Google 항목의 "웹 SDK 구성" 클라이언트 ID가 GCP에서 수정한 클라이언트와 일치하는지 확인 (여러 OAuth 클라이언트가 있을 경우 다른 걸 수정했을 수 있음)

### momcal.vercel.app 서비스워커 이슈 — "지금 방식 유지"로 결정됨
momcal.vercel.app 접속자가 거의 없어서, 301 리다이렉트(momcal.vercel.app → momcal.app)는 그대로 유지하기로 결정. 참고로 momcal.vercel.app에 예전에 설치된 서비스워커는 리다이렉트 때문에 더 이상 자체 업데이트를 받을 수 없어(브라우저가 서비스워커 스크립트 fetch에서 리다이렉트를 따라가지 않음) 영구적으로 예전 버전에 머무름 — 이건 코드로 고칠 수 없고 해당 사용자가 브라우저에서 사이트 데이터를 지워야만 해결됨. 접속자가 적다는 판단하에 그냥 두기로 함.

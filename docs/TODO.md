### 알아두면 좋은 것
- **`js/modal.js`의 `cm()`은 모달을 닫을 때 `#modal`의 `open` 클래스만 뗄 뿐 `#mB`(모달 본문) 내용은 지우지 않음(v0.3.0에서 재확인)** — CSS(`.mo.open { display:flex }`)로 안 보이게만 할 뿐, 마지막으로 열렸던 모달의 HTML(입력했던 id들 포함)은 닫은 뒤에도 다른 모달이 `#mB`를 새로 채우기 전까지 DOM에 그대로 남아있음. 그래서 "특정 모달 안 요소(`getElementById`)가 DOM에 있는지"로 "그 모달이 지금 열려있는지"를 판단하면 틀림(v0.2.5의 정부지원 삭제 버그 재발 원인) — 반드시 `document.getElementById('modal')?.classList.contains('open')`처럼 `#modal` 자체의 `open` 클래스도 함께 확인할 것. 근본적으로 `cm()`이 닫을 때 `#mB.innerHTML`도 비우게 고치면 이 클래스의 버그를 원천 차단할 수 있는데, 다른 모달 흐름에 영향이 있을 수 있어 아직 미적용(v0.3.0 시점, 옹짐꾼님 확인 필요)\n- 정식 도메인은 `momcal.app` — `momcal.vercel.app`도 301 리다이렉트로 계속 접속은 가능(단, 서비스워커 이슈로 예전 방문자는 갱신 안 될 수 있음, 감수하기로 결정됨)

## 다음 후보 (미정 — 우선순위 논의 필요)

> v0.0.16부터 "고민 중" 항목도 이 섹션에서 함께 관리합니다.

- [ ] **홈 화면에 "오늘 해야 할 일" 요약 추가** — 오늘 날짜 기준 캘린더 일정·체크리스트 미완료 필수 항목을 한 곳에 모아 보여주는 카드. 지금은 "다음 일정" 카드가 있지만 "오늘 안에 뭘 해야 하는지"를 한눈에 모아 보여주진 않음
- [ ] **체크리스트 항목 전면 추가/수정** — 옹짐꾼님이 세부 내용 주시면 진행
- [ ] **"이 설명이 도움됐나요?" 피드백 기능 유지/삭제 결정 필요** — 지금은 계정별 개인 저장(`S.itemFeedback`, guide 페이지는 localStorage)이라 다른 사용자에게 아무 영향이 없는 장식용 버튼에 가까움. 계속 유지한다면 전체 사용자 집계로 바꿔서(예: 항목별 "도움됐어요 N명" 카운트를 Firestore에 모아 보여주는 방식) 실제 유용한 정보로 만들지, 아니면 기능 자체를 없앨지 결정 필요
- [ ] 병원 방문 기록 상세 (진료과·의사 메모 등)
- [ ] AI 육아비서 구조 (OpenAI 연결 준비)
- [ ] 원더윅스(Wonder Weeks) 기간·발달 현황 기간을 캘린더에 표시하는 기능 — v0.0.42에서 만든 "캘린더 연동 켜기/끄기가 실제로 일정을 캘린더에 보였다 숨겼다 하는" 구조(`js/calendar.js`의 `getAutoEvs()` 끝부분 필터, `S.clSettings.calendarSync`)를 그대로 재사용할 수 있음 — 새 카테고리를 하나 만들고 그 계산 로직을 `getAutoEvs()`에 추가한 뒤 같은 방식으로 켜기/끄기 토글만 붙이면 됨
- [ ] 임신 체크리스트 주차별 세분화 분류 (필수/추천/정부지원/병원/준비물/육아팁)
- [ ] 정부지원 항목별 정확한 신청 링크(딥링크)로 교체
- [ ] 제도 정보(금액·자격 요건) 연 1회 이상 최신화 필요

## 디자인 철학 (v0.0.20 개정 — "AI스럽지 않은 실제 서비스 수준의 UI/UX")

이전엔 "따뜻하고 귀여운 톤"을 파스텔 다색 + 장식용 이모지로 표현했는데, 실사용 피드백으로 **"AI가 만든 티가 난다"**는 지적을 받아 톤을 다시 잡았습니다. 파스텔/이모지 자체가 문제가 아니라 **"장식이 정보보다 시끄러운 것"**이 문제였다는 결론 — 이번 개정은 색과 이모지를 줄이고, 타이포그래피와 여백으로 위계를 만드는 방향입니다.

- **팔레트: 핑크 · 화이트 · 그레이만 사용** — 기존의 민트/보라/파랑/노랑은 "장식"이 아니라 **정보를 구분하는 용도(캘린더 카테고리, 등급 배지, 테마 선택지)일 때만** 예외로 유지합니다. 새 UI 요소를 만들 때 색을 고민된다면 기본값은 핑크 아니면 그레이입니다 — 컬러가 하나 더 필요하다고 느껴지면 "이게 정말 서로 다른 정보를 나타내는가?"를 먼저 자문할 것 (아래 "색상 예외 목록" 참고)
- **이모지 대부분 삭제 → 아이콘 라이브러리로 대체** — UI 크롬(내비게이션·버튼·카드·섹션 라벨·설정 항목)의 이모지는 [Material Symbols](https://fonts.google.com/icons)(Outlined) 아이콘으로 교체합니다. `js/utils.js`의 `icon(name, opts)` 헬퍼로만 아이콘을 만드세요 — 마크업을 직접 쓰지 말 것(아래 "아이콘 시스템" 참고). **예외**: 캘린더 스티커(사용자가 직접 고르는 콘텐츠), 아이 프로필 아바타(👦/👧/🍼), 등급 배지 텍스트에 곁들이는 아이콘은 이모지가 아니라 이미 아이콘으로 전환됨 — 남은 이모지는 "장식" 목적이 아니라 "그 자체가 사용자 콘텐츠"인 경우로 한정
- **둥글기 16px** — 기존 20px보다 살짝 각지게. `--radius: 16px`. 작은 요소(배지·태그)는 별도 값 유지
- **여백은 12 / 16 / 20px 스케일로** — `--sp-sm`(12px) / `--sp-md`(16px) / `--sp-lg`(20px) 세 단계만 쓰고, 그때그때 임의의 padding 값(예: 13px, 15px, 18px)을 새로 만들지 마세요. 기존 컴포넌트의 애매한 값들은 발견하는 대로 이 세 값 중 가장 가까운 것으로 정리
- **그라디언트는 한 곳에만** — 이전엔 버튼·활성 탭·배지·배너에 전부 같은 핑크→보라 그라디언트를 복붙해서 "어디가 정말 중요한지" 위계가 사라졌습니다. 지금은 **메인 CTA 버튼(`.bpk`) 한 곳에만** 그라디언트를 남기고, 나머지 강조(활성 탭, 진행률 바 등)는 단색 `var(--pk)`를 씁니다
- **모바일 우선** — 손가락으로 누르기 쉬운 터치 타겟 (최소 44px)
- **과하지 않은 애니메이션** — 0.2~0.3s ease, 필요한 곳만

### "AI 티" 자가 진단 — 새 UI를 만들고 나서 체크할 것
1. **같은 그라디언트를 이 화면에 몇 번 썼는가?** 2번 이상이면 위계가 없다는 신호 — 하나만 남기고 나머지는 단색으로
2. **카드마다 아이콘 배경색이 다른가?** (파스텔 무지개 사각형들) — 정보 구분 목적이 아니라면 전부 같은 중립색(`var(--gray-100)`)으로 통일
3. **장식용 이모지가 텍스트 앞뒤에 붙어있는가?** (예: "✨ 완료했어요!", "🎉 축하해요!") — 삭제하거나 `icon()` 아이콘으로 교체
4. **여백 값이 12/16/20 중 하나가 아닌가?** — 그렇다면 가장 가까운 값으로 반올림
5. **문구가 상투적인가?** ("함께해요", "더 나은 내일을 위해" 같은 두루뭉술한 감탄사) — 구체적인 정보로 대체 (예: "완료했어요!" → "3/5 완료")

### 색상 예외 목록 (팔레트 규칙에서 제외되는 곳 — "정보 전달용"이라 유지)
| 위치 | 색 | 이유 |
|---|---|---|
| 캘린더 이벤트 카테고리(필수/추천/접종/정부지원) | 민트·보라·파랑·그린 | 일정 종류를 한눈에 구분하는 정보 — 장식 아님 |
| 캘린더 테마 선택지(rose/mint/sunny/lavender/peach) | 5색 | 사용자가 직접 고르는 개인화 옵션 |
| 체크리스트 등급 배지(Perfect/Master/Legend) | 금색/보라/무지개 그라디언트 | 게임화 보상 — 등급마다 다른 색이 성취감을 줌 |
| 위험/삭제 동작 | 빨강(`#E53935`류) | "destructive" 예외 — 다른 스택에서도 흔히 유지하는 예외 |

## 컬러 시스템

모든 색상은 CSS 변수로 사용. **하드코딩 금지 — 이 규칙이 지켜지지 않아서 다크 모드 미대응 요소가 버전마다 재발했음** (`docs/CHANGELOG.md` 참고). 새 색이 필요하면 이 파일에 변수를 먼저 추가하고 코드에서는 변수만 참조할 것.

```css
:root {
  /* 메인 팔레트 — 핑크·화이트·그레이 (v0.0.20부터 기본값) */
  --pk:  #F06292;   /* 버튼, 강조, 선택 */
  --pkl: #FFF0F5;   /* 핑크 배경 (연함) */
  --pkd: #C2185B;   /* 핑크 텍스트 (진함) */
  --gray-50:  #FAFAFA;
  --gray-100: #F2F2F5;  /* 카드 안 아이콘 배경 등 중립 배경 */
  --gray-200: #E5E5EA;
  --gray-300: #D1D1D8;
  --gray-500: #8A849A;  /* = --txl */
  --gray-700: #4A4A57;

  /* 아래는 "정보 전달용" 예외 색 — 위 "색상 예외 목록" 참고, 장식엔 쓰지 말 것 */
  --mn:  #4DB6AC; --mnl: #E0F2F1;   /* 캘린더: 추천 */
  --pu:  #9575CD; --pul: #EDE7F6;   /* 캘린더: 접종 */
  --bl:  #64B5F6; --bll: #E3F2FD;   /* 캘린더: 내 일정(기본값, 개별 색은 사용자가 고름) */
  --yl:  #FFD54F; --yll: #FFFDE7;   /* 배지: Perfect 등급 */
  --pk2: #CE93D8;                   /* 배지: Master 등급 */

  /* 텍스트 */
  --tx:  #2D2D3A;   /* 본문 */
  --txl: #8A849A;   /* 보조 */
  --wh:  #ffffff;

  /* 배경 */
  --bg:   #F8F4FA;
  --card: #ffffff;
  --card-shadow: 0 2px 16px rgba(160,80,140,.09);

  /* 반경·간격 (v0.0.20) */

  --sp-sm: 12px;
  --sp-md: 16px;
  --sp-lg: 20px;
}
```

### 그라디언트 패턴 (v0.0.20 — 메인 CTA 한 곳으로 축소)

```css
/* 메인 버튼(.bpk)에만 — 앱에서 유일하게 그라디언트를 쓰는 자리 */

/* 그 외 강조(활성 탭, 진행률 바, 사이드바 헤더 등)는 전부 단색 */
```

## 아이콘 시스템 (v0.0.20 신설)

**Material Symbols Outlined**를 씁니다 (Google Fonts CDN, `index.html`/`guide/*.html`/`privacy.html`/`terms.html`/`contact.html` 전부 `<head>`에 로드되어 있음). 새 아이콘이 필요하면 [fonts.google.com/icons](https://fonts.google.com/icons)에서 이름만 찾아서 쓰면 됩니다 — 별도 설치 불필요.

### 사용법
JS 모듈에서는 **반드시 `js/utils.js`의 `icon()` 헬퍼를 통해서만** 아이콘을 만드세요 (마크업을 직접 쓰지 말 것 — "아이콘 색은 훅 하나로 통일" 원칙, 나중에 라이브러리를 바꾸거나 스타일을 조정할 때 이 함수 하나만 고치면 되게 하기 위함):

```js
import { icon } from './utils.js';

icon('home')                          // 기본 24px
icon('checklist', { size: 'sm' })     // 작게(네비게이션 등 좁은 공간)
icon('trending_up', { size: 'lg' })   // 크게
icon('verified', { fill: true })      // 채워진 스타일(활성 상태 등)
```

정적 HTML(`index.html`의 고정 마크업)에서는 직접 이 형태로 쓰면 됩니다:
```html
<span class="icon icon-sm" translate="no" aria-hidden="true">calendar_month</span>
```

색은 항상 `currentColor`를 상속하므로 아이콘 자체에 색을 지정하지 말고, **부모 요소의 `color`만 바꾸면 아이콘 색도 함께 바뀝니다.**

### 예외 (아이콘으로 바꾸지 않고 이모지 그대로 유지)
- **캘린더 스티커** — 사용자가 직접 고르는 콘텐츠 자체가 이모지임(`data/stickerCats`, `js/calendar.js`의 `stickerCats`). UI 크롬이 아니라 사용자가 만든 데이터라서 아이콘화 대상이 아님. 스티커 탭 헤더(`label`)는 아이콘으로 바꿨지만, 탭 안의 개별 스티커(`items`)는 그대로 이모지
- **아이 프로필 아바타**(👦/👧/🍼) — 아이콘보다 친근한 "아바타" 성격이 강해 유지
- **예방접종(💉)·정부지원(🟢) 마커** — `js/calendar.js`의 `getAutoEvs()`가 자동 일정 제목 앞에 `💉 `/`🟢 `를 붙여서 저장하고, 이 접두어가 `getEventKey()`·회차 자동 재계산(`vaccineSeries.js`)·체크리스트 연동(`checklistCalendarLink.js`)의 문자열 매칭에 그대로 쓰임 — **UI 장식이 아니라 데이터 포맷**이라 아이콘으로 바꾸면 여러 기능이 한꺼번에 깨짐. 건드리지 말 것
- **체크리스트 카테고리 라벨의 의미있는 이모지** — 임신 체크는 실제 태아 크기(과일 비교), 육아 체크는 씨앗→나무→아이로 자라나는 성장 진행감을 표현하려고 의도적으로 고른 이모지라 아이콘으로 바꾸지 않음(`data/checklist-data.js`, 아래 "카테고리 라벨" 참고)
- **등급 배지 텍스트 옆 보조 표현** — 배지는 아이콘(`verified`/`workspace_premium`/`auto_awesome`)으로 이미 전환됨, 색으로 등급을 구분하는 것은 위 "색상 예외 목록" 참고

### 카테고리 라벨 (체크리스트 — 아이콘화 예외)
- **임신 체크리스트(9단계)**: 각 주차 구간을 실제 태아 크기와 비슷한 과일로 표현 — 🫐(4~7주,블루베리) → 🍇(8~11주,포도) → 🥝(12~15주,키위) → 🥑(16~19주,아보카도) → 🥭(20~23주,망고) → 🍆(24~27주,가지) → 🥥(28~31주,코코넛) → 🍍(32~35주,파인애플) → 🍉(36~40주,수박·만삭). 호주 정부 보건 자료(Pregnancy Birth & Baby)의 표준 주차별 크기 비교를 기준으로 정함
- **육아 체크리스트(9단계)**: 씨앗에서 나무로, 다시 아이로 자라나는 성장 진행감을 표현 — 🌰(0~1개월,씨앗) → 🌱(2~3개월,새싹) → 🌿(4~5개월) → 🍀(6~8개월) → 🪴(9~11개월,화분) → 🌳(12~17개월,나무) → 👶(18~23개월,아기) → 🧒(24~35개월,어린이) → 🧑(36~60개월,다 자란 아이)
- 두 항목 모두 `data/checklist-data.js`의 `label` 필드에 있고, 여기 있는 이모지를 바꾸면 앱 체크리스트·`guide/pregnancy.html`·`guide/parenting.html` 양쪽에 자동으로 반영됨(단일 소스)

### 마이그레이션 현황 (v0.0.21 기준 — 거의 완료)
UI 크롬 이모지는 하단 내비게이션, 등록/로그인 버튼, 홈 대시보드 카드, 체크리스트/정부지원 배지·아이콘, 캘린더 일정 수정 모달, 데이 패널, 스티커 탭 헤더, 알림·다크모드·글자크기 설정 행, 계정삭제·가족공유 모달, 성장 기록 모달, 육아정보(`guide/`) 페이지 전체(히어로·카드·네비게이션·CTA·검색창)까지 대부분 아이콘으로 전환 완료. 남은 것은 위 "예외" 목록(스티커·아바타·💉🟢 마커·카테고리 라벨)뿐이며, 새로 발견되는 장식용 이모지가 있으면 그때그때 `icon()` 헬퍼로 정리하면 됨.

## 다크 모드 (v0.0.5, v0.0.8에서 육아정보 페이지까지 확대)

설정 탭(`js/theme.js`)에서 켜고 끌 수 있음. `<html data-theme="dark">` 속성으로 전환되고, `css/main.css`(앱 본체)·`guide/guide.css`(육아정보 페이지, v0.0.8부터) 안의 `html[data-theme="dark"] { ... }` 블록이 색상을 재정의함. 기기별 `localStorage`에 저장되며 Firestore에는 저장되지 않음(로그인해도 다른 기기와 동기화되지 않음).

- **적용 범위**: v0.0.8부터 앱 본체(`index.html`)와 육아정보 페이지(`guide/`) 모두 지원. 단, 육아정보 페이지엔 다크 모드를 켜고 끄는 토글이 따로 없음 — 앱 설정 탭에서 고른 값을 각 페이지의 `<head>` 인라인 스크립트(`scripts/build-guide.mjs`의 `head()`, `privacy.html`/`terms.html`/`contact.html`도 동일)가 읽어서 그대로 적용만 함
- **작동 원리**: 위 "컬러 시스템"의 CSS 변수(`--bg`/`--card`/`--tx`/`--txl`/`--wh` 등)를 다크용 값으로 재정의 — 이 변수들을 쓰는 요소는 대부분 자동으로 전환됨
- **한계**: "컬러 시스템" 원칙상 색상은 변수로 써야 하지만, 실제 코드에는 `#fff`/`#EEE5F4`/`#FBF6FA`/`#FAFAFA` 같은 하드코딩된 색이 곳곳에 남아있어서 이런 요소들은 `html[data-theme="dark"] .클래스 { ... }` 형태로 하나씩 별도로 덮어써야 했음. v0.0.5에서 topbar·nav pill·로그인 화면 등 1차로 덮었고, v0.0.8에서 실사용 캡처를 받아 `.ad-fallback`(홈 육아 팁), `.ci`(체크리스트 미완료 행), `.gov-cl-item`(정부지원 체크리스트), `.growth-record-item`(성장 기록 목록), `.auth-tabs`, `.progress-bar`, `.sp-tab`(스티커 탭) 등을 추가로 덮었음. v0.0.9에서 다시 캡처를 받아 가족 프로필 카드(`.pcard.sel`/`.add-pcard`), 체크리스트 탭 버튼(`.cl-tab-btn`), 체크리스트 컨텍스트 배너(`.cl-context-banner`), 체크리스트 배지 3종(`.cl-sb-perfect`/`.cl-sb-master`/`.cl-sb-legend`, `.cl-badge-perfect`/`-master`/`-legend`), 정부지원 상태 배지(`.gov-cl-status`), 성장그래프 안내문구·빈 상태 카드(`.growth-disclaimer`/`.growth-pct-empty`), 저장 완료 배지(`.save-badge`)까지 추가로 덮음. v0.0.15에서는 설정 탭 가족 그룹 공유 카드의 "가족 그룹 나가기" 버튼(`#FFF5F5` 인라인 배경)을 `.family-leave-btn` 클래스로 분리해 덮음. **새 컴포넌트를 만들 때 하드코딩된 색 대신 CSS 변수를 쓰면, 다크 모드가 별도 작업 없이 자동으로 적용됨** — 이 원칙을 지키는 게 왜 중요한지 보여주는 사례. 그럼에도 매 버전마다 놓친 곳이 계속 발견되는 걸 보면, 아직 다 찾지 못한 하드코딩 색이 더 있을 가능성이 높음
- **광고 관련 주의**: `.ad-fallback`(AdSense 미연동 시 대신 보여주는 육아 팁)은 우리가 만든 컴포넌트라 다크 대응이 가능했지만, 나중에 실제 Google AdSense 광고(`AD_ENABLED = true`)가 붙으면 그 광고 iframe 자체는 우리가 스타일을 바꿀 수 없어서 다크 모드에서도 흰 배경으로 보일 수 있음(정상, 손댈 수 없는 영역)
- 새로운 화면/컴포넌트를 추가했는데 다크 모드에서 이상하게 보이면(밝은 배경에 밝은 글씨 등), 앱 본체는 `css/main.css`, 육아정보 페이지는 `guide/guide.css`의 `html[data-theme="dark"]` 블록에 해당 클래스를 추가해서 보정할 것 — **두 파일 다 확인**해야 함(핵심 개발 원칙 9번)

## 타이포그래피

```css
/* 제목·본문·버튼·"맘캘" 로고 — 전부 동일 (v0.0.3부터) */
font-family: 'OwnglyphParkDahyun', 'Apple SD Gothic Neo', 'Noto Sans KR', sans-serif;
font-weight: 400; /* Regular만 존재 — Bold 파일 없음 */
```

- **v0.0.3부터 폰트 하나로 통일**: 옹짐꾼님이 제공한 손글씨 폰트 "온글잎 박다현체"(Ownglyph PDH)를 제목·본문·버튼·로고 구분 없이 전부 사용. 이전처럼 로고/제목/본문/버튼마다 다른 폰트를 쓰던 방식(Paperlogy+Pretendard)을 완전히 그만둠
- **v0.0.4→v0.0.5 예외 — "정보 글"은 별도 손글씨 폰트**: 체크리스트 세부 설명(`.ci-detail`)·육아정보 항목 설명(`.g-item p`)·정책 페이지 본문(`.g-doc p`/`li`)처럼 정보 전달이 목적인 긴 글은 제목/본문에 쓰는 온글잎 박다현체 대신 옹짐꾼님이 주신 또 다른 손글씨 폰트 "오뮤 다예쁨체"(Omyu Pretty)를 씀 — v0.0.4에서 잠깐 Pretendard로 원복했다가, v0.0.5에서 "폰트가 따로 노는 느낌"이라는 피드백으로 이 폰트로 교체(둘 다 손글씨 계열이라 톤이 더 잘 어울림). **새로운 "정보성 긴 글" UI를 만들 땐 이 폰트를 따를 것** — 제목/버튼/로고/짧은 라벨은 계속 온글잎 박다현체
- **파일 위치**: `fonts/OwnglyphParkDahyun.ttf`(제목·본문·버튼·로고), `fonts/OmyuPretty.ttf`(정보 글) — 둘 다 외부 CDN이 아니라 프로젝트에 직접 포함된 파일이며, `css/main.css`와 `guide/guide.css`에 각각 `@font-face`로 선언되어 있음(두 파일이 별도 스타일시트라 폰트를 바꿀 땐 항상 같이 수정 필요)
- **두 폰트 모두 Bold 파일 없음**: Regular 굵기만 제공됨. 브라우저가 임의로 두껍게 그리는 faux bold를 방지하기 위해, 원래 Bold였던 제목(`h1~h3`/`.sec`)·로고(`.logo`/`.auth-logo`/`.g-logo`)·버튼(`.btn`/`.st-btn`/`.g-btn`/`.np`)도 전부 `font-weight: normal`로 맞춰둠. 손글씨체 특성상 안 굵어도 다른 텍스트와 충분히 구분됨
- **한글 글자 수 제한**: 두 폰트 다 실사용 빈도가 높은 한글 위주로만 지원함(전체 한글 11,172자보다 적음). 희귀한 한자어·이름 등 특수 글자를 쓰면 해당 글자만 시스템 기본 폰트로 대체되어 보일 수 있음 — 실제로 깨지는 곳이 보이면 캡처해서 알려주면 확인 가능
- 라이선스: 온글잎 박다현체·오뮤 다예쁨체 둘 다 영리적·상업적 사용 포함 모든 용도에 제한 없이 무료 사용 가능(단, 폰트 자체의 재판매·재배포·수정은 금지 — 저작권은 각각 (주)보이저엑스/오뮤다이어리에 있음). AdSense 수익화 목적에도 문제 없음
- **v0.0.4 — 기준 글자 크기 확대**: `html { font-size: 17px }`(기존 브라우저 기본 16px)로 앱 전체 `rem` 기반 글자가 함께 커짐. 단, 캘린더 셀 안 이벤트 텍스트(`.ev-line`/`.ev-more`)는 셀 공간이 빠듯해 과거 여러 버전(Sprint 25·27)에 걸쳐 맞춰둔 크기라 `px`로 고정해 이 확대에서 제외함 — 캘린더 날짜 숫자(`.day-num`)는 예외 없이 다른 곳과 동일하게 커짐
- **v0.0.4 — 로고 크기와 마스코트 크기 분리**: 이전엔 `.brand-mark`(마스코트 이미지) 높이가 로고 글씨 크기에 상대적인 `em` 단위였어서, 글씨만 키우려 해도 마스코트까지 같이 커지는 구조였음 — 마스코트 크기를 고정 `rem` 값으로 분리해서 글씨(`.logo`/`.g-logo` 1.5rem→2.2rem)만 독립적으로 키울 수 있게 정리. 로그인 화면(`.auth-logo`)은 이 변경의 영향을 받지 않도록 별도 예외 처리되어 있음
- ⚠️ **앱 본체(`css/main.css`)와 육아정보 페이지(`guide/guide.css`)는 별도 파일이라 폰트 규칙이 자동으로 동기화되지 않음** — 폰트를 바꿀 땐 항상 두 파일(`@font-face` 선언 포함)을 함께 수정할 것 (PROJECT_SPEC.md "핵심 개발 원칙" 9번 참고)

## 컴포넌트 규칙

### 카드 (`.card`)

```css
background: var(--card);
border-radius: var(--radius);   /* 16px, v0.0.20에서 20px→16px */
box-shadow: var(--card-shadow);
padding: var(--sp-lg);           /* 20px */
```

- 카드 안 카드는 `border-radius: 12px`, `padding: var(--sp-md)`(16px)
- hover 시: `transform: translateY(-3px)` + 그림자 강화

### 버튼

```css
/* 메인 (bpk) — 앱에서 그라디언트를 쓰는 유일한 자리 */

border-radius: 13px;
font-weight: 800;
padding: var(--sp-sm) var(--sp-lg);   /* 12px 20px */

/* 완료/확인(bmn)을 포함해 그 외 버튼은 전부 단색 */
background: var(--mn);   /* 완료/확인 — 이전엔 그라디언트였음(v0.0.20에서 단색으로) */

/* hover 공통 */
opacity: .87;
transform: translateY(-1px);
```

```css
background: var(--gray-100);
```

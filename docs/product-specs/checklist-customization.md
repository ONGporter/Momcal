# 체크리스트 커스터마이징 (v0.0.40)

> 상태: ✅ 코드 완료, 실기기 확인 대기 (별도 배포 절차 없음 — 다른 앱 본체 기능과 함께 Vercel 자동 배포)

## 배경

기존 체크리스트(예방접종/발달/이유식/정부지원)는 전부 **월령·주차로 인덱싱된** 구조였음. 앞으로 옹짐꾼님이 계속 추가할 예정인 카테고리(외출 준비물, 100일 준비, 돌 준비, 돌사진 준비 등)는 월령 인덱싱이 없는 **"한 번에 쭉 확인하는" 플랫 리스트**라서, 이 둘을 같은 "탭" 개념으로 묶되 렌더링은 통일했음. 탭이 계속 늘어날 걸 감안해서 "원하는 탭만 보기" · "캘린더 연동 여부 선택" · "직접 만들기"를 이번에 함께 넣음.

## 탭 종류

| kind | 예시 | 구조 |
|---|---|---|
| `indexed` | 임신 체크·출산 준비물·예방접종·발달·이유식 | 월령/주차별 하위 카테고리 여러 개 (기존 구조 그대로) |
| `gov` | 정부지원 | `js/govSupport.js`가 별도 렌더링 (체크리스트 아님, 대상 제도 목록) |
| `flat` | 준비물 팩(외출 준비물 등) · 사용자 정의 체크리스트 | 하위 카테고리 없이 카테고리 1개짜리 — `getCats()`가 `[{key, label, items}]` 형태로 감싸서 반환하면 기존 `calcScore`/`getCatItems`/`renderClMain`이 그대로 처리함(신규 렌더 코드 없음) |

## 핵심 설계 — "키 기반" 탭 판별

기존엔 `S.clTab`(숫자 인덱스)이 "0=예방접종, 1=발달…"처럼 **위치가 고정**돼 있다고 가정하고 `renderContextBanner`/`autoSelectCat`/`getCats`/`getCustomKey`가 전부 숫자로 분기했음. 탭을 숨길 수 있게 되면 위치가 사용자마다/순간마다 달라지므로, 이제 이 함수들은 전부 아래 방식으로 판별함:

```js
const tabDefs = getVisibleTabDefs(child); // 지금 보이는 탭 목록
const curKey  = tabDefs[S.clTab || 0]?.key; // 'vax' | 'dev' | 'food' | 'gov' | 'preg' | 'prep' | pack/custom key
```

`S.clTab`은 Firestore에 저장되지 않는 세션 전용 값(`js/state.js`의 `emptyState()`에서 초기화만 되고 `saveState()`엔 포함 안 됨)이라, 이 의미 변경이 기존 저장 데이터에 영향을 주지 않음.

## 데이터 모델 (Firestore, additive)

```json
{
  "customChecklists": [
    { "id": 1720000000000, "key": "custom_1720000000000", "label": "어린이집 준비물", "icon": "checklist", "stage": "born",
      "items": [{ "id": "cclt_1720000000000_0", "t": "가방", "r": true }] }
  ],
  "clSettings": {
    "hiddenTabs": ["food"],
    "calendarSync": { "vax": false }
  }
}
```

- `customChecklists`: 사용자가 "새 체크리스트 만들기"로 만든 플랫 체크리스트. 항목 id는 `cclt_`로 시작(기존 "＋ 항목 직접 추가하기"가 만드는 `custom_` 접두어와 겹치면 삭제 버튼이 엉뚱한 배열을 건드리게 됨 — 반드시 접두어를 분리할 것). `stage`(`'preg'`|`'born'`, v0.0.41 추가)가 없으면 `'born'`으로 취급(v0.0.40에서 만든 기존 데이터 하위 호환)
- `clSettings.hiddenTabs`: 숨긴 탭의 key 배열(내장 6종 + 팩 4종 + 커스텀 공용). 전부 숨기면 `getVisibleTabDefs()`가 안전장치로 전체를 다시 보여줌
- `clSettings.calendarSync`: 명시적으로 끈 것만 저장(기본은 켜짐) — `{ vax: false }`처럼 값이 있을 때만 꺼진 것으로 취급

## 준비물 팩 — `data/checklist-packs.js`

내장 6종(외출 준비물/100일 준비/돌 준비/돌사진 준비/태명 정하기/태교여행 준비)이 들어있는 배열. 새 팩을 추가하려면 이 배열에 `{ key, label, icon, stage, items }` 객체 하나만 추가하면 됨 — 렌더링 코드는 건드릴 필요 없음(`js/checklist.js`의 `packTabDefs()`가 자동으로 탭 목록에 포함시킴). `stage` 필드(`'preg'` | `'born'`, 기본값 `'born'`)로 아이 단계에 맞는 팩만 노출됨. `dd`(상세 설명) 필드는 앱의 "자세히 보기" 아코디언과 `scripts/build-guide.mjs`가 생성하는 guide 페이지 양쪽에 그대로 반영됨(v0.0.41).

## 캘린더 연동 켜기/끄기

`js/checklistCalendarLink.js`가 실제로 캘린더와 항목을 연결하는 곳은 **오직 예방접종(vax)·발달(dev) 탭**뿐임(`data/checklist-links.js`에 명시적으로 매핑된 항목만). 이유식·정부지원·준비물 팩·커스텀 체크리스트는 애초에 이 매핑 파일에 항목이 없어서 캘린더에 아무것도 만들지 않음 — 그래서 설정 화면에도 이 두 탭에만 "캘린더 연동" 토글을 보여줌(있지도 않은 연동을 끄는 척하는 UI를 만들지 않기 위한 의도적 선택).

토글을 끄면 `syncChecklistToCalendar`/`syncCalendarToChecklist`가 조기 `return false`로 아무 것도 하지 않음 — 체크리스트 체크와 캘린더 완료 표시가 서로 독립적으로 동작하게 됨(양쪽 다 계속 별도로 쓸 수 있음, 그냥 서로 자동으로 맞춰주지 않을 뿐).

## UI 진입점

- **설정 탭 → "체크리스트 관리"** — 전체 관리 화면. v0.0.41부터 **임산부용 / 육아용 / 공통(정부지원)** 세 그룹으로 나눠서 보여줌(내장 6종이 한 목록에 섞여있어 헷갈린다는 피드백 반영). 표시/숨김, 캘린더 연동, 새로 만들기, 커스텀 편집/삭제가 전부 여기 있음
- **체크리스트 탭 바 오른쪽 톱니바퀴 아이콘** — 설정 탭으로 바로 이동하는 바로가기(중복 UI를 만들지 않기 위해 실제 관리 기능은 설정 탭에만 있음)
- **커스텀 체크리스트 편집(v0.0.41)** — 설정 탭의 각 커스텀 체크리스트 행 옆 연필 아이콘. 이름·항목(텍스트/필수여부/추가/삭제)을 한 화면에서 통째로 고칠 수 있음. 기존 항목은 id를 그대로 유지해서 그동안 체크한 기록이 끊기지 않고, 새로 추가한 행만 새 id(`cclt_{체크리스트id}_{timestamp}_{순번}`)를 받음

## 알려진 한계 (다음 후보)

- 탭 순서를 사용자가 직접 바꾸는 기능(드래그 정렬)은 없음 — 항상 내장 → 팩 → 커스텀 순서로 고정
- 준비물 팩·커스텀 체크리스트를 캘린더에 표시하는 기능은 없음(원래 목적이 "캘린더를 지저분하게 하지 않는 것"이라 의도적으로 없음) — 필요해지면 별도 논의 필요
- guide 페이지에 반영된 준비물 팩 콘텐츠는 앱 체크리스트와 `dd` 필드를 그대로 공유하므로, 내용을 고치려면 `data/checklist-packs.js` 한 곳만 수정하고 `node scripts/build-guide.mjs`를 재실행하면 됨(둘이 따로 관리되지 않음)
